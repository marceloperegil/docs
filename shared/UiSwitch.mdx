# Componente `UiSwitch`

<Callout type="info">
  Este documento describe el componente Blazor `UiSwitch`, un control de interfaz de usuario para aplicaciones Blazor.
</Callout>

## Propósito

El componente `UiSwitch` proporciona un control de interfaz de usuario de tipo interruptor (toggle switch) con una etiqueta configurable. Su objetivo principal es ofrecer una manera intuitiva y visualmente atractiva para que los usuarios alternen entre dos estados (por ejemplo, activado/desactivado, sí/no) dentro de una aplicación Blazor.

## Explicación Funcional

El `UiSwitch` es un componente Blazor reutilizable que encapsula la lógica y la presentación de un interruptor moderno, comúnmente visto en sistemas operativos y aplicaciones web. Está diseñado para ser sencillo de integrar y de usar con el enlace de datos bidireccional de Blazor.

Internamente, el componente renderiza un elemento `<input type="checkbox" role="switch">` al que se le aplican clases CSS de Bootstrap para transformarlo en un switch visual.

### Parámetros

| Nombre           | Tipo                  | Descripción                                                                                                                                                                                          |
| :--------------- | :-------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Checked`        | `bool`                | **(Input)** Obtiene o establece el estado actual del interruptor. `true` indica que el interruptor está activado, `false` indica que está desactivado. Es la parte "get" del enlace de datos bidireccional. |
| `CheckedChanged` | `EventCallback<bool>` | **(Output)** Un `EventCallback` que se invoca cuando el usuario cambia el estado del interruptor. Pasa el nuevo valor booleano (`true` o `false`) al componente padre. Es crucial para el enlace de datos bidireccional (`@bind-Checked`). |
| `Label`          | `string`              | **(Input)** El texto que se muestra junto al interruptor, sirviendo como su etiqueta descriptiva. Si no se proporciona, se usará una cadena vacía.                                                            |

### Comportamiento

Cuando el usuario interactúa con el interruptor y cambia su estado (al hacer clic o tocarlo), el método `OnChange` del componente se encarga de:

1.  Capturar el `ChangeEventArgs` del evento HTML.
2.  Extraer de forma robusta el valor booleano del estado del interruptor. Esto maneja tanto los casos en los que `e.Value` es directamente un `bool` como cuando es una representación de cadena ("true"/"false").
3.  Invocar el `EventCallback` `CheckedChanged` con el nuevo valor booleano. Esto notifica al componente padre sobre el cambio y permite que el enlace de datos bidireccional (`@bind-Checked`) actualice la propiedad correspondiente en el padre.

## Dependencias

### Framework
*   **Blazor:** Es un componente Blazor nativo y requiere un entorno de ejecución Blazor (WebAssembly o Server).

### Estilos
*   **Bootstrap 5 (o superior):** El estilo visual del interruptor (`form-switch`, `form-check-input`, `form-check-label`) depende directamente de las clases CSS proporcionadas por Bootstrap. Es fundamental que Bootstrap esté correctamente incluido y referenciado en tu proyecto Blazor (normalmente en `wwwroot/index.html` para Blazor WebAssembly, o `_Layout.cshtml` / `App.razor` para Blazor Server).

### Otras Librerías
*   No tiene dependencias directas de librerías JavaScript adicionales ni de otros componentes Blazor personalizados más allá del propio framework Blazor y Bootstrap para estilos.

## Ejemplo de Uso

Para utilizar el componente `UiSwitch`, incorpóralo en cualquier componente Blazor padre. El patrón de uso más común es a través del enlace de datos bidireccional `@bind-Checked` para vincular el estado del switch a una propiedad booleana en tu componente padre.

```razor title="MySettingsComponent.razor"
@page "/configuracion"

<h3>Preferencias de Usuario</h3>

<div class="mb-3">
    <UiSwitch @bind-Checked="isDarkModeEnabled" Label="Activar Modo Oscuro" />
    <p class="mt-2">Modo Oscuro: <strong>@(isDarkModeEnabled ? "Activado" : "Desactivado")</strong></p>
</div>

<div class="mb-3">
    <UiSwitch @bind-Checked="receiveNewsletter" Label="Recibir Boletín Mensual" />
    <p class="mt-2">Suscripción a Boletín: <strong>@(receiveNewsletter ? "Sí" : "No")</strong></p>
</div>

<div class="mb-3">
    <UiSwitch @bind-Checked="showAlerts" Label="Mostrar Alertas de Notificación" />
    <p class="mt-2">Alertas: <strong>@(showAlerts ? "Activadas" : "Desactivadas")</strong></p>
</div>

@code {
    private bool isDarkModeEnabled = true;
    private bool receiveNewsletter = false;
    private bool showAlerts = true;
}
```

En este ejemplo:

*   Las propiedades `isDarkModeEnabled`, `receiveNewsletter`, y `showAlerts` son variables `bool` definidas en el componente padre (`MySettingsComponent`).
*   La sintaxis `@bind-Checked="propertyName"` establece un enlace bidireccional:
    *   El valor inicial de `Checked` del `UiSwitch` se toma de `propertyName`.
    *   Cuando el usuario cambia el `UiSwitch`, el `EventCallback` `CheckedChanged` se invoca y Blazor actualiza automáticamente `propertyName` con el nuevo valor.
*   La propiedad `Label` se utiliza para proporcionar una descripción clara y concisa para cada interruptor, mejorando la usabilidad.
