# Componente UiDatePicker

El `UiDatePicker` es un componente Blazor reutilizable diseñado para facilitar la selección de fechas por parte del usuario, encapsulando la lógica de entrada y salida, y proporcionando una interfaz limpia para otros componentes.

## Propósito

El propósito principal de este componente es:

1.  **Simplificar la Entrada de Fechas:** Ofrecer una interfaz de usuario estándar para que los usuarios seleccionen una fecha.
2.  **Encapsular la Lógica de Parsing:** Manejar la conversión de la entrada de texto del usuario a un objeto `DateTime?` de forma segura, incluyendo el manejo de entradas inválidas.
3.  **Facilitar el Binding Bidireccional:** Permitir una comunicación fluida de ida y vuelta con componentes padres que necesiten establecer o reaccionar a la fecha seleccionada.

## Explicación Funcional

El componente `UiDatePicker` se construye alrededor del componente `InputDate` nativo de Blazor, lo que significa que aprovecha la funcionalidad de entrada de fecha del navegador (generalmente un calendario o selector de fecha).

### Interfaz de Usuario (UI)

*   Rendersiza un `<input type="date">` a través del componente `InputDate` de Blazor.
*   Aplica la clase CSS `form-control` para asegurar una estilización básica consistente, típicamente usada en frameworks como Bootstrap.

```razor filename="LirmiDemoApp.Client/Shared/UiDatePicker.razor"
<InputDate @bind-Value="SelectedDate" class="form-control" @onchange="OnDateChangedHandler" />
```

### Lógica de Componente (`@code`)

El componente expone dos parámetros clave y una lógica para manejar los cambios:

1.  **`SelectedDate` (Parámetro de Entrada/Salida):**
    *   `[Parameter] public DateTime? SelectedDate { get; set; }`
    *   Este parámetro permite que un componente padre le pase una fecha inicial (o `null`) al `UiDatePicker`.
    *   Gracias a la convención de Blazor (`@bind-Value` emparejado con `EventCallback<T> NombreDePropiedadChanged`), también actúa como la propiedad de la cual el padre recibirá las actualizaciones. El `?` indica que la fecha es opcional (nullable).

2.  **`SelectedDateChanged` (Callback de Evento):**
    *   `[Parameter] public EventCallback<DateTime?> SelectedDateChanged { get; set; }`
    *   Este es un `EventCallback` que el componente invoca cuando la fecha seleccionada por el usuario cambia.
    *   Es fundamental para la funcionalidad de binding bidireccional de Blazor, permitiendo que el componente notifique a su padre sobre los cambios en `SelectedDate`.

3.  **`OnDateChangedHandler` (Manejador de Eventos):**
    *   `private async Task OnDateChangedHandler(ChangeEventArgs e)`
    *   Este método se ejecuta cada vez que el valor del `InputDate` cambia (por ejemplo, cuando el usuario selecciona una fecha del calendario o la escribe).
    *   **Parsing Seguro:** Utiliza `DateTime.TryParse` para intentar convertir la cadena de texto (`e.Value?.ToString()`) a un objeto `DateTime`. Esta es una práctica robusta para evitar errores si el usuario introduce un formato de fecha no válido o texto arbitrario.
    *   **Notificación al Padre:**
        *   Si el parsing es exitoso, invoca `SelectedDateChanged.InvokeAsync(date)` con la fecha parseada.
        *   Si el parsing falla (la entrada no es una fecha válida), invoca `SelectedDateChanged.InvokeAsync(null)` para indicar al componente padre que la fecha debe ser limpiada o establecida como `null`.

```csharp filename="LirmiDemoApp.Client/Shared/UiDatePicker.razor"
@code {
    [Parameter] public DateTime? SelectedDate { get; set; }
    [Parameter] public EventCallback<DateTime?> SelectedDateChanged { get; set; }

    private async Task OnDateChangedHandler(ChangeEventArgs e)
    {
        if (DateTime.TryParse(e.Value?.ToString(), out var date))
        {
            await SelectedDateChanged.InvokeAsync(date);
        }
        else
        {
            await SelectedDateChanged.InvokeAsync(null);
        }
    }
}
```

## Dependencias

### Framework

*   **Microsoft.AspNetCore.Components:** Blazor framework components (`InputDate`, `[Parameter]`, `EventCallback`, `ChangeEventArgs`).
*   **System:** Tipos de .NET (`DateTime?`, `Task`).

### Externas

*   Ninguna dependencia externa o paquete NuGet adicional fuera del framework Blazor estándar.

## Ejemplo de Uso

Para utilizar el `UiDatePicker` en un componente padre, simplemente se necesita una propiedad `DateTime?` en el componente padre y se utiliza la sintaxis de binding bidireccional de Blazor:

```razor filename="ParentComponent.razor"
@page "/date-selection"

<h3>Selección de Fecha de Nacimiento</h3>

<div class="form-group">
    <label>Fecha de Nacimiento:</label>
    <UiDatePicker @bind-SelectedDate="fechaNacimiento" />
</div>

<p>La fecha seleccionada es: @(fechaNacimiento?.ToShortDateString() ?? "Ninguna")</p>

@code {
    private DateTime? fechaNacimiento { get; set; } = DateTime.Now; // Inicializar con la fecha actual, o null
}
```

<Callout emoji="💡">
  La clave del binding bidireccional es la convención Blazor: el parámetro `SelectedDate` y el `EventCallback SelectedDateChanged` se emparejan automáticamente al usar `@bind-SelectedDate` en el componente padre. Esto permite que el componente padre pase un valor y reciba actualizaciones sin lógica adicional.
</Callout>
