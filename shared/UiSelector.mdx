# Componente UiSelector

Este documento describe el funcionamiento y uso del componente Blazor `UiSelector`, diseñado para proporcionar un control de selección (dropdown) reutilizable y de fácil integración.

---

## Propósito

El `UiSelector` es un componente Blazor diseñado para:

*   **Abstraer la lógica de un `<select>` HTML:** Encapsula la creación y gestión de un control de selección estándar.
*   **Proporcionar un control reutilizable:** Permite definir un selector de opciones que puede ser utilizado en múltiples lugares de la aplicación con diferentes conjuntos de datos.
*   **Facilitar la selección de opciones:** Ofrece una interfaz simple para que los usuarios elijan un elemento de una lista predefinida.
*   **Implementar binding bidireccional:** Permite que el componente padre pase un valor inicial y sea notificado automáticamente cuando el usuario cambia la selección.

---

## Explicación Funcional

El componente `UiSelector` es un wrapper alrededor del componente `InputSelect` de Blazor, lo que le permite manejar la mayoría de las interacciones de un elemento `<select>` de forma nativa.

### 1. Definición de Opciones

El componente recibe una lista de opciones a través del parámetro `Options`. Cada opción es una instancia de la clase anidada `OptionItem`, que se compone de dos propiedades:
*   `Value`: El valor interno asociado a la opción (por ejemplo, un ID o un código).
*   `Text`: El texto que se mostrará al usuario en el dropdown.

El bucle `@foreach` se encarga de renderizar dinámicamente cada `<option>` HTML dentro del `<InputSelect>`, utilizando el `Value` y `Text` de cada `OptionItem`.

### 2. Binding de Valor (Bidireccional)

El componente utiliza un patrón de binding bidireccional para manejar el valor seleccionado:
*   **`SelectedValue` (Parameter):** Recibe el valor actualmente seleccionado del componente padre. Si este valor cambia programáticamente en el padre, el `UiSelector` actualizará su selección visual.
*   **`SelectedValueChanged` (EventCallback):** Es un `EventCallback` que notifica al componente padre cuando el usuario ha cambiado la selección en el `UiSelector`. El nombre `SelectedValueChanged` es una convención de Blazor que permite el uso de la sintaxis `@bind-SelectedValue` en el componente padre.

### 3. Manejo de Eventos

Cuando el usuario selecciona una nueva opción, el evento `@onchange` del `InputSelect` dispara el método `OnValueChanged`. Este método:
1.  Extrae el nuevo valor seleccionado de los argumentos del evento (`e.Value?.ToString()`).
2.  Invoca asincrónicamente el `EventCallback` `SelectedValueChanged`, pasando el nuevo valor al componente padre.

De esta manera, el `UiSelector` mantiene su interfaz limpia y su estado sincronizado con el componente que lo utiliza.

### 4. Estilo

El componente incluye la clase `form-select`, que es una clase de Bootstrap. Esto le da una apariencia estándar de un control de selección dentro de un formulario Bootstrap, aunque puede ser sobrescrito o ignorado si la aplicación usa otra biblioteca CSS o estilos personalizados.

---

## Dependencias

### Framework de Blazor
*   **`Microsoft.AspNetCore.Components`**: Necesario para el uso de `[Parameter]`, `EventCallback<T>`, `ChangeEventArgs` y el componente `InputSelect`.
*   **`System.Collections.Generic`**: Para el uso de `List<T>`.

### Estilos (Opcional)
*   **Bootstrap**: La clase `form-select` sugiere una dependencia de la biblioteca CSS de Bootstrap para el estilo predeterminado del control de selección. El componente funcionará sin Bootstrap, pero su apariencia puede variar.

---

## Ejemplo de Uso

Para utilizar el `UiSelector` en un componente Blazor padre, se deben seguir los siguientes pasos:

1.  **Definir la propiedad `SelectedValue`** en el componente padre para almacenar la selección actual.
2.  **Crear una lista de `UiSelector.OptionItem`** para las opciones disponibles.
3.  **Renderizar el `UiSelector`**, pasando la lista de opciones y utilizando `@bind-SelectedValue` para la comunicación bidireccional.

```razor filename="Pages/MyParentComponent.razor"
@page "/parent-selector"

<h3>Ejemplo de Uso del UiSelector</h3>

<div class="mb-3">
    <label for="mySelector" class="form-label">Seleccione una opción:</label>
    <UiSelector id="mySelector"
                @bind-SelectedValue="currentSelection"
                Options="availableCountries">
    </UiSelector>
</div>

<p>La opción seleccionada actualmente es: <strong>@currentSelection</strong></p>

@code {
    private string currentSelection = "ES"; // Valor inicial

    private List<LirmiDemoApp.Client.Shared.UiSelector.OptionItem> availableCountries = new();

    protected override void OnInitialized()
    {
        // Simulamos la carga de opciones al inicializar el componente
        availableCountries.Add(new LirmiDemoApp.Client.Shared.UiSelector.OptionItem { Value = "US", Text = "Estados Unidos" });
        availableCountries.Add(new LirmiDemoApp.Client.Shared.UiSelector.OptionItem { Value = "CA", Text = "Canadá" });
        availableCountries.Add(new LirmiDemoApp.Client.Shared.UiSelector.OptionItem { Value = "MX", Text = "México" });
        availableCountries.Add(new LirmiDemoApp.Client.Shared.UiSelector.OptionItem { Value = "ES", Text = "España" });
        availableCountries.Add(new LirmiDemoApp.Client.Shared.UiSelector.OptionItem { Value = "AR", Text = "Argentina" });
    }

    // Opcional: Si necesitas realizar alguna acción específica cuando cambia la selección
    // puedes suscribirte al evento SelectedValueChanged directamente, aunque @bind-SelectedValue
    // ya maneja la actualización de currentSelection.
    // private void HandleSelectionChanged(string newValue)

    //     Console.WriteLine($"El usuario seleccionó: {newValue}");
    // }
}
```
En este ejemplo, el `MyParentComponent` inicializa el `UiSelector` con una lista de países y establece "ES" como la selección inicial. Cuando el usuario cambia la opción en el dropdown, la propiedad `currentSelection` en `MyParentComponent` se actualizará automáticamente.
