# Componente `NavMenu.razor`: Barra de Navegación Global

El `NavMenu.razor` es el componente fundamental para la navegación dentro de la aplicación LirmiDemoApp. Proporciona una barra lateral (o "hamburger menu" en móviles) que permite a los usuarios acceder a las diferentes secciones de la aplicación, gestionar su sesión de usuario y ver información clave como el título de la aplicación.

## Propósito

El propósito principal de este componente es:

1.  **Facilitar la Navegación:** Ofrecer un conjunto claro de enlaces a las rutas principales de la aplicación.
2.  **Identificación de la Aplicación:** Mostrar el título de la aplicación (`LirmiDemoApp`).
3.  **Experiencia de Usuario Adaptativa:** Implementar un menú responsivo que se colapsa y expande, adecuado para diferentes tamaños de pantalla.
4.  **Gestión de Autenticación/Autorización:** Adaptar dinámicamente las opciones de navegación según el estado de autenticación del usuario (logueado o no logueado).
5.  **Mantenimiento del Estado de Navegación:** Resaltar el enlace de la página activa y gestionar el `ReturnUrl` para el cierre de sesión.

## Explicación Funcional

El componente `NavMenu.razor` está construido sobre una estructura HTML que utiliza clases de Bootstrap para su estilizado y responsividad.

### Estructura de Diseño

*   **Barra Superior (`.top-row`):** Contiene el título de la aplicación (`LirmiDemoApp`) y sirve como un elemento siempre visible.
*   **Toggle de Navegación (`.navbar-toggler`):** Un `input` de tipo `checkbox` que funciona como un "botón de hamburguesa" para ocultar/mostrar el menú de navegación en dispositivos móviles. La lógica de clic se gestiona mediante JavaScript simple (`document.querySelector('.navbar-toggler').click()`) en el contenedor principal del menú (`.nav-scrollable`) para cerrar el menú al hacer clic fuera de él.
*   **Área de Navegación (`.nav-scrollable`):** Contiene todos los `NavLink` y se desplaza si hay muchos elementos.

### Enlaces de Navegación Estáticos

El menú incluye una serie de enlaces fijos que dirigen a secciones principales de la aplicación:

*   **Home:** Ruta raíz (`/`), con `Match="NavLinkMatch.All"` para asegurar que solo se active en la página de inicio exacta.
*   **Counter:** Ruta `/counter`.
*   **Weather:** Ruta `/weather`.
*   **Auth Required:** Ruta `/auth`.
*   **Demo Tabla:** Ruta `/demo-table`.

Cada `NavLink` utiliza iconos de Bootstrap (ej. `bi bi-house-door-fill-nav-menu`) para una mejor experiencia visual.

### Lógica de Autenticación y Autorización (`<AuthorizeView>`)

Una de las características clave es la adaptación del menú según el estado de autenticación del usuario, utilizando el componente `<AuthorizeView>` de Blazor:

*   **Usuario Autenticado (`<Authorized>`):**
    *   Muestra el nombre del usuario (`@context.User.Identity?.Name`) con un enlace a la página de administración de cuenta (`Account/Manage`).
    *   Presenta un formulario de "Logout" (cierre de sesión) que es un `POST` a `Account/Logout`.
    *   Incluye un `<AntiforgeryToken />` para protección contra ataques CSRF.
    *   Un campo `hidden` llamado `ReturnUrl` que almacena la URL actual de la aplicación (`@currentUrl`), permitiendo redirigir al usuario a la página desde la que inició el logout.

*   **Usuario No Autenticado (`<NotAuthorized>`):**
    *   Muestra enlaces a las páginas de "Register" (`Account/Register`) y "Login" (`Account/Login`).

### Gestión de Estado en el Bloque `@code`

El componente implementa `IDisposable` y utiliza `NavigationManager` para mantener el `currentUrl` actualizado:

*   **`currentUrl`:** Una propiedad privada que almacena la ruta relativa base de la URL actual. Es crucial para el `ReturnUrl` del formulario de logout.
*   **`OnInitialized()`:** Se ejecuta una vez cuando el componente es inicializado.
    *   Obtiene la URL inicial utilizando `NavigationManager.ToBaseRelativePath(NavigationManager.Uri)`.
    *   Se suscribe al evento `NavigationManager.LocationChanged` para detectar cambios de URL.
*   **`OnLocationChanged(object? sender, LocationChangedEventArgs e)`:** Se invoca cada vez que la URL de navegación cambia.
    *   Actualiza `currentUrl` con la nueva ubicación.
    *   Llama a `StateHasChanged()` para forzar el renderizado del componente y asegurar que el `ReturnUrl` en el formulario de logout esté siempre actualizado.
*   **`Dispose()`:** Implementación del método de la interfaz `IDisposable`.
    *   Desuscribe el componente del evento `NavigationManager.LocationChanged` para prevenir fugas de memoria cuando el componente es destruido.

## Dependencias

Este componente tiene las siguientes dependencias clave:

*   **`NavigationManager`:** Un servicio inyectado (`@inject NavigationManager`) que permite interactuar programáticamente con la navegación de Blazor, obtener la URL actual y suscribirse a cambios de ubicación.
*   **`IDisposable`:** Una interfaz implementada para gestionar la liberación de recursos (desuscripción de eventos) cuando el componente es destruido.
*   **`NavLink` (Componente Blazor):** Componente integrado de Blazor que representa un enlace de navegación y automáticamente aplica una clase "active" cuando la URL actual coincide con el `href` del enlace.
*   **`AuthorizeView` (Componente Blazor):** Componente integrado de Blazor que permite mostrar contenido condicionalmente basándose en el estado de autenticación y autorización del usuario actual.
*   **`AntiforgeryToken` (Blazor Feature):** Un componente o `tag helper` utilizado dentro de formularios POST para proteger contra ataques de falsificación de solicitudes entre sitios (CSRF).
*   **Bootstrap (CSS):** El componente utiliza numerosas clases CSS de Bootstrap para su diseño y responsividad (ej. `navbar`, `nav-item`, `flex-column`, `navbar-toggler`, `bi bi-...`).

## Ejemplo de Uso

Dado que `NavMenu.razor` es un componente de diseño (layout), su uso principal es incrustarlo en el componente de diseño principal de la aplicación, típicamente `MainLayout.razor` o `App.razor`.

**`MainLayout.razor` (Ejemplo de integración):**

```razor filename="MainLayout.razor" {3}
@inherits LayoutComponentBase

<div class="page">
    <div class="sidebar">
        <NavMenu />
    </div>

    <main>
        <div class="top-row px-4">
            <a href="https://docs.microsoft.com/aspnet/core/" target="_blank">About</a>
        </div>

        <article class="content px-4">
            @Body
        </article>
    </main>
</div>
```

En este ejemplo, el componente `<NavMenu />` se coloca dentro del `div` con la clase `sidebar`, lo que lo posiciona como la barra de navegación lateral de la aplicación.
