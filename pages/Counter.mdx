## Propósito

La página `Counter` es un componente de Blazor que permite a los usuarios interactuar con un contador simple. Este ejemplo es comúnmente utilizado para demostrar conceptos básicos de Blazor, como el manejo de eventos y el estado del componente. Es útil en aplicaciones donde se necesita mostrar una interacción básica y directa con el usuario.

## Arquitectura y Funcionamiento

La página `Counter` sigue un flujo sencillo que incluye:

### Ciclo de vida

Este componente no utiliza métodos específicos del ciclo de vida de Blazor como `OnInitialized` o `OnParametersSet`. Su funcionalidad se basa únicamente en el manejo de eventos y el estado interno.

### Lógica de renderizado condicional

El contenido de la página se renderiza de forma estática, excepto por el valor del contador (`currentCount`), que se actualiza dinámicamente cada vez que el usuario hace clic en el botón. Blazor se encarga de actualizar automáticamente el DOM cuando cambia el estado.

### Manejo de estado

- **Variable interna:** `currentCount` es una variable privada que almacena el estado actual del contador.
- **Actualización del estado:** Cada vez que se invoca el método `IncrementCount`, el valor de `currentCount` se incrementa en 1, lo que provoca un re-renderizado del contenido asociado.

### Manejo de eventos

- **Evento de clic:** El botón está vinculado al método `IncrementCount` mediante el atributo `@onclick`. Cada vez que el usuario hace clic en el botón, se ejecuta este método.

### Flujo de datos

- **Entrada:** No se reciben parámetros externos.
- **Transformación:** El método `IncrementCount` modifica el estado interno del componente.
- **Salida:** El valor actualizado de `currentCount` se muestra dinámicamente en el párrafo.

## Dependencias

- **Namespaces:** No se requiere ningún namespace adicional en este componente.
- **Componentes hijos:** No utiliza componentes hijos.
- **Servicios inyectados:** No se inyectan servicios en este componente.
- **Paquetes NuGet:** No se utilizan paquetes externos.

## Ejemplo de Uso

<CodeGroup>
<CodeGroup.Entry label="Básico">
```razor
@page "/counter"
@rendermode InteractiveAuto
@*Actualizar*@
<PageTitle>Counter</PageTitle>

<h1>Counter page</h1>

<p role="status">Current count: @currentCount</p>

<button class="btn btn-primary" @onclick="IncrementCount">Click me</button>

@code {
    private int currentCount = 0;

    private void IncrementCount()
    {
        currentCount += 1;
    }
}
```
</CodeGroup.Entry>
</CodeGroup>

## Notas Técnicas

<Tip>
Para mejorar el rendimiento, asegúrate de que el componente no realice operaciones costosas en cada renderizado. En este caso, el contador es ligero y no afecta significativamente el rendimiento.
</Tip>

<Warning>
Este componente no maneja límites para el contador. Si necesitas restringir el valor máximo o mínimo, deberás agregar lógica adicional.
</Warning>

<Note>
Es una buena práctica utilizar nombres descriptivos para las variables y métodos, como `currentCount` y `IncrementCount`, para facilitar la comprensión del código.
</Note>
