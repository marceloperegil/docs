# Página de Demostración del Toolkit de UI (`/ui-toolkit`)

## Propósito

La página `/ui-toolkit` tiene como propósito principal servir de escaparate interactivo y guía de uso para los componentes de interfaz de usuario personalizados (`UiAutocomplete`, `UiSelector`, `UiCheckbox`, `UiSwitch`, `UiDatePicker`, `UiDateRangePicker`) disponibles en la aplicación `LirmiDemoApp.Client`. Permite a los desarrolladores y usuarios visualizar su funcionalidad y comportamiento en un entorno real, facilitando la comprensión de su integración y manejo de eventos.

## Explicación Funcional

Esta página es una `Razor Page` de Blazor que renderiza una serie de componentes de UI personalizados, cada uno en una sección independiente. Para cada componente, se demuestra cómo enlazar sus propiedades (valores, opciones) y cómo manejar sus eventos de cambio de estado.

A continuación, se detalla la funcionalidad de cada sección:

### 1. Autocomplete (`UiAutocomplete`)

*   **Descripción:** Un componente que permite seleccionar un elemento de una lista predefinida mediante búsqueda predictiva (autocompletado).
*   **Estado:** La página mantiene una lista de ítems (`autocompleteItems`) y una variable para el valor seleccionado (`selectedAutocomplete`).
*   **Interacción:** Al seleccionar un valor en el `UiAutocomplete`, el método `OnAutocompleteSelected` se invoca para actualizar `selectedAutocomplete` y forzar un `StateHasChanged()` para reflejar el nuevo valor en la UI.
*   **Propiedades clave:**
    *   `Items`: La lista de opciones disponibles para el autocompletado.
    *   `Value`: El valor actualmente seleccionado o enlazado.
    *   `ValueChanged` / `OnSelected`: Eventos que se disparan cuando cambia el valor seleccionado.

### 2. Selector (`UiSelector`)

*   **Descripción:** Un componente desplegable (dropdown) que permite seleccionar una opción de una lista predefinida de objetos con `Value` y `Text`.
*   **Estado:** La página define una lista de objetos `UiSelector.OptionItem` (`selectorOptions`) y una variable para el valor seleccionado (`selectedSelector`).
*   **Interacción:** Cuando se selecciona una opción, el método `OnSelectorChanged` actualiza `selectedSelector` y la UI.
*   **Propiedades clave:**
    *   `Options`: La lista de opciones disponibles, donde cada opción es un objeto con `Value` y `Text`.
    *   `SelectedValue`: El valor de la opción actualmente seleccionada.
    *   `SelectedValueChanged`: Evento que se dispara cuando cambia la opción seleccionada.

### 3. Checkbox (`UiCheckbox`)

*   **Descripción:** Un componente de casilla de verificación para toggling de un estado booleano.
*   **Estado:** La página mantiene una variable booleana (`checkboxChecked`) para el estado del checkbox.
*   **Interacción:** Al marcar o desmarcar el checkbox, el método `OnCheckboxChanged` actualiza `checkboxChecked` y la UI.
*   **Propiedades clave:**
    *   `Checked`: El estado actual (marcado/desmarcado) del checkbox.
    *   `CheckedChanged`: Evento que se dispara cuando el estado `Checked` cambia.
    *   `Label`: Texto descriptivo asociado al checkbox.

### 4. Switch (`UiSwitch`)

*   **Descripción:** Un componente de interruptor (toggle switch) similar a un checkbox, pero con una representación visual diferente.
*   **Estado:** La página mantiene una variable booleana (`switchChecked`) para el estado del switch.
*   **Interacción:** Al activar o desactivar el switch, el método `OnSwitchChanged` actualiza `switchChecked` y la UI.
*   **Propiedades clave:**
    *   `Checked`: El estado actual (activado/desactivado) del switch.
    *   `CheckedChanged`: Evento que se dispara cuando el estado `Checked` cambia.
    *   `Label`: Texto descriptivo asociado al switch.

### 5. Selector de Fecha (`UiDatePicker`)

*   **Descripción:** Un componente para la selección de una única fecha del calendario.
*   **Estado:** La página mantiene una variable `DateTime?` (`selectedDate`) para la fecha seleccionada.
*   **Interacción:** Al seleccionar una fecha, el método `OnDateChanged` actualiza `selectedDate` y la UI.
*   **Propiedades clave:**
    *   `SelectedDate`: La fecha actualmente seleccionada. Puede ser `null`.
    *   `SelectedDateChanged`: Evento que se dispara cuando la fecha seleccionada cambia.

### 6. Selector de Rango de Fechas (`UiDateRangePicker`)

*   **Descripción:** Un componente para la selección de un rango de fechas (fecha de inicio y fecha de fin).
*   **Estado:** La página mantiene dos variables `DateTime?` (`startDate` y `endDate`) para las fechas de inicio y fin del rango.
*   **Interacción:** Al cambiar la fecha de inicio o fin, los métodos `OnStartDateChanged` y `OnEndDateChanged` respectivamente, actualizan las variables correspondientes y la UI.
*   **Propiedades clave:**
    *   `StartDate`: La fecha de inicio del rango actualmente seleccionada.
    *   `StartDateChanged`: Evento que se dispara cuando la `StartDate` cambia.
    *   `EndDate`: La fecha de fin del rango actualmente seleccionada.
    *   `EndDateChanged`: Evento que se dispara cuando la `EndDate` cambia.

## Ejemplo de Uso

La propia página `UiToolkitPage.razor` es el ejemplo de uso más directo de estos componentes. A continuación, se muestra un fragmento representativo del uso del `UiAutocomplete` junto con su lógica de soporte en el bloque `@code`, demostrando el patrón de enlace de datos y manejo de eventos que se aplica a todos los componentes de la página.

```csharp showLineNumbers
// LirmiDemoApp.Client/Pages/UiToolkitPage.razor

@page "/ui-toolkit"

@using LirmiDemoApp.Client.Shared

<h3>UI Toolkit Demo</h3>

<div class="mb-4">
    <h5>Autocomplete</h5>
    {/* Integración del componente UiAutocomplete */}
    <UiAutocomplete Items="autocompleteItems" Value="@selectedAutocomplete" ValueChanged="@OnAutocompleteSelected" OnSelected="@OnAutocompleteSelected" />
    <div>Seleccionado: @selectedAutocomplete</div>
</div>

@code {
    // Autocomplete
    private List<string> autocompleteItems = new() { "Chile", "Argentina", "Perú", "Colombia", "Brasil", "Ecuador", "Uruguay", "Paraguay", "Bolivia", "Venezuela" };
    private string selectedAutocomplete;

    // Manejador de eventos para el Autocomplete
    private Task OnAutocompleteSelected(string value) {
        selectedAutocomplete = value;
        StateHasChanged(); // Notifica a Blazor que el estado ha cambiado para que la UI se actualice
        return Task.CompletedTask;
    }

    // ... otros componentes y su lógica
}
```

Este patrón de:
1.  Declarar una variable de estado en el bloque `@code`.
2.  Pasar esta variable como propiedad `Value` (o similar, como `Checked`, `SelectedDate`, `StartDate`, `EndDate`) al componente.
3.  Crear un método `On...Changed` que actualice la variable de estado y llame a `StateHasChanged()`.
4.  Enlazar este método al evento `ValueChanged` (o similar, como `CheckedChanged`, `SelectedDateChanged`, `StartDateChanged`, `EndDateChanged`) del componente.

Es consistente a lo largo de toda la página para cada uno de los componentes de UI demostrados.
