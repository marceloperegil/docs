# Componente Blazor: `Counter.razor`

Este documento detalla el componente `Counter.razor`, un ejemplo fundamental en Blazor para demostrar la interactividad básica del lado del cliente.

## Propósito

El componente `Counter.razor` sirve como una demostración simple de un contador interactivo. Muestra cómo:
- Definir una ruta para una página Blazor.
- Utilizar el modo de renderizado interactivo para habilitar la interactividad del lado del cliente.
- Gestionar el estado de un componente con variables C#.
- Responder a eventos de usuario (clics de botón) para actualizar el estado y la interfaz de usuario.
- Actualizar dinámicamente el contenido HTML basado en el estado del componente.

Es un componente "hola mundo" para entender los conceptos básicos de Blazor interactivo en el contexto de una Blazor Web App (anteriormente conocido como Blazor United).

## Explicación Funcional

El componente `Counter` consta de una sección de UI (HTML con Blazor directives) y una sección de lógica (`@code` C#).

### 1. Definición de Ruta

```razor
@page "/counter"
```
La directiva `@page "/counter"` especifica que este componente actúa como una página enrutada y es accesible directamente a través de la URL `/counter` en la aplicación Blazor.

### 2. Modo de Renderizado Interactivo

```razor
@rendermode InteractiveAuto
```
La directiva `@rendermode InteractiveAuto` es crucial. Indica a Blazor que renderice este componente de forma interactiva en el cliente.
-   **`Interactive`**: Habilita la interactividad del lado del cliente (ej. manejo de eventos de clic).
-   **`Auto`**: Blazor intentará usar Blazor Server primero (para una carga inicial más rápida y una experiencia sin Javascript para el usuario final en la primera carga) y, si es necesario o configurado, recurrirá a Blazor WebAssembly, optimizando el tiempo de carga inicial y la persistencia de la conexión. Sin esta directiva, el componente se renderizaría estáticamente del lado del servidor y no respondería a los clics del botón.

<Callout type="info" title="Importancia del Rendermode">
    El `rendermode` es fundamental para la arquitectura de Blazor Web Apps (Blazor United). Permite mezclar componentes estáticos y estáticamente renderizados con componentes que requieren interactividad del lado del cliente (WebAssembly) o del servidor (Server), eligiendo el modo de renderizado óptimo para cada parte de la aplicación.
</Callout>

### 3. Interfaz de Usuario (UI)

```html
<PageTitle>Counter</PageTitle>

<h1>Counter</h1>

<p role="status">Current count: @currentCount</p>

<button class="btn btn-primary" @onclick="IncrementCount">Click me</button>
```
-   **`<PageTitle>Counter</PageTitle>`**: Establece el título de la pestaña del navegador para esta página.
-   **`<h1>Counter</h1>`**: Un encabezado estándar que presenta el propósito de la página.
-   **`<p role="status">Current count: @currentCount</p>`**: Muestra el valor actual de la variable `currentCount`. El `@` indica que `currentCount` es una expresión C# que se evaluará y su resultado se insertará en el HTML. El atributo `role="status"` se utiliza para mejorar la accesibilidad, indicando a los lectores de pantalla que este elemento es un área en vivo que puede actualizarse.
-   **`<button class="btn btn-primary" @onclick="IncrementCount">Click me</button>`**: Un botón HTML con estilos de Bootstrap (`btn btn-primary`). La directiva `@onclick="IncrementCount"` vincula el evento de clic del botón al método C# `IncrementCount` definido en el bloque `@code`. Cada vez que se hace clic en el botón, se ejecuta este método.

### 4. Lógica del Componente (C#)

```csharp
@code {
    private int currentCount = 0;

    private void IncrementCount()
    {
        currentCount += 6;
    }
}
```
-   **`private int currentCount = 0;`**: Declara una variable privada entera llamada `currentCount` y la inicializa en `0`. Esta variable es el *estado* del componente y se utiliza para almacenar el valor actual del contador.
-   **`private void IncrementCount()`**: Este método se invoca cuando el usuario hace clic en el botón.
    -   `currentCount += 6;`: Incrementa el valor de `currentCount` en 6.
    -   Después de ejecutar este método, Blazor detecta que el estado del componente ha cambiado y automáticamente vuelve a renderizar las partes afectadas de la UI (en este caso, la etiqueta `<p>` que muestra `currentCount`) para reflejar el nuevo valor. Esto es parte del ciclo de vida automático de Blazor.

## Dependencias

Este componente tiene dependencias mínimas y se considera en gran medida autocontenido para su funcionalidad principal:

-   **Core Blazor Framework**: Es una parte integral de cualquier aplicación Blazor (en este caso, una Blazor Web App).
-   **Proyecto Cliente Blazor (`LirmiDemoApp.Client`)**: Se ejecuta dentro del contexto de este proyecto, que gestiona el host de WebAssembly o Server.
-   **Bootstrap (Opcional, para estilos)**: La clase `btn btn-primary` en el botón sugiere el uso del framework CSS Bootstrap para la estilización. Si Bootstrap no está incluido o enlazado en el proyecto, el botón aparecerá con estilos por defecto del navegador, pero la funcionalidad seguirá siendo la misma.
-   No se requieren paquetes NuGet externos o servicios inyectados adicionales para su funcionalidad básica.

## Ejemplo de Uso

Para interactuar con este componente una vez que la aplicación Blazor está en ejecución:

1.  Asegúrese de que la aplicación Blazor (`LirmiDemoApp`) esté iniciada y accesible.
2.  Abra su navegador web y navegue a la siguiente URL (reemplazando `your-app-domain` con la dirección de su aplicación, por ejemplo, `localhost:5000`):
    `https://your-app-domain/counter`
3.  Una vez en la página, haga clic en el botón "Click me". Observará cómo el contador aumenta en 6 con cada clic, demostrando la reactividad de Blazor.
